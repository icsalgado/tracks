no terminal do vscode
	dotnet new console - ele cria o projeto na pasta
	dotnet run - para executar o código dentro da pasta

Sintaxe e tipos de dados em C#
	program.cs > é a classe 
	.csproj > arquivo de projeto do c# - informações descritivas do projeto, xml com metadados
		output type se é um executavel ou bibliotecas
		target framework versão do .net usada [é possível mudar a versão]
			no cli > dotnet --info > informa as versões instaladas mas uma versão nova pode quebrar o projeto
		os demais são referentes a como o projeto deve se comportar
	
	diretorio obj > maior parte dos arquivos são referentes ao debugger e compilação, raramente é preciso manipula-la
	diretorio bin > os binários dll, exe, pdb(podem ser usados para debug)
		esses dois diretórios são gerados com > dotnet build > podem ser ignoradas para novas compilações

	Conceito de Classe:
		New folder > new C2 file > class
			nome da classe PascalCase
				prop > atalho para criar uma propriedade
				para criar um novo metodo
					public void Apresentar(){}

			o namespace não necessáriamente representa o caminho físico, é uma boa prática usar identificações relevantes e relacionadas
			O @ pode identificar o uso de uma palavra reservado para uma outra finalidade mas não é indicado. Ex: string @class
			
			ao instanciar a pessoa dentro do Program é preciso indicar o namespace onde está a classe pessoa
				instanciar é criar uma variavel, um objeto do tipo da classe desejada. é constuido o objeto com a base da classe
				Ex: using SintaxeEtipos.Models
				também é possível instanciar usando o namespace porém ficará mais verboso
					ex: SintaxeEtipos.Models.Pessoa pes = new SintaxeEtipos.Models.Pessoa
	Cases
		padrões para escrita - camelCase, PascalCase, snake_case, spinal-case
		classes > PascalCase
		propriedades > PascalCase
		metodo > PascalCase
		variavel > camelCase
	Escrita de nomes
		sem abreviações
		nome da classe deve ser o mesmo do nome do arquivo.cs
		sem caracteres especiais, exceto _
