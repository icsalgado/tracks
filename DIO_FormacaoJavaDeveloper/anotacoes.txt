Java - Anatomia das classes
	ctrl + shift + p > Java: Create Java Project > No build tools
		forma do nome do projeto: java-anatomia-classes
		forma do nome de classe: PascalCase

	o nome da classe deve ser igual ao nome do arquivo (MinhaClasse.java >> public class MinhaClasse {})

		variaveis são camelCase
			variaveis que nunca vão ter o valor alterado devem ser em UpperCase ex: final String BR = "Brasil";
				o final traz a garantia da imutabilidade mas o programador deve saber que não  deve ser alterada
		variaveis podem ter ou iniciar _ , $ ou letras 

	Java é fortemente tipado e o valor da variavel pode ser atribuido no momento da declaração ou depois
	
	Metodos:
		TipoDeRetorno NomeNoInfinitivo Parametros()
			ex: int somar (int num1, int num2)
				pode receber parametros de diferentes tipos e retornar um outro tipo
	
	Packages > diretórios para organização
		dominio reverso
			ex: edu.iuri.aula
		No vscode
			criar o pacote um nível por vez
				edu
					iuri
						aula
			E aí então pode ser movida a classe para dentro do aula

	Java Beans:
		estruturação de escrita proposta pela comunidade
		Variaveis sempre no singular, com idioma único, evitar abreviação do sentido
		Metodos devem ser nomeados como verbo ex: somar, somarInteiros

Tipos e Variaveis
	tipos primitivos (valores brutos não armazenados na pila de memoria)
		int (-2147483648 a 2147483647), byte (-128 a 127), short (-32768 a 32767), long (8 bytes), float, double, boolean, char
	Peculiaridades, valores inteiros que começam com zero, talvez precise usar uma string
		float prefisa encerrar o valor com um F e o long com L
		cuidado com valores com precisão maior atribuidos a variaveis de precisão menor
	constantes
		escritas em UPPER CASE
			final tipo-da-variavel nome-da-variavel ex: final float PI = 3.14F;

Operadores
	= usado para atribuição ou alteração de valor a uma variável
	! operador unário lógico de negação
	Ternário
		<condicional> ? <verdadeira> : <falsa>;

	.equals vai comparar o conteúdo entre duas variáveis

Métodos, parâmetros e visibilidade
	uma classe é definida por atributos e métodos, que são ações ou sub-rotinas
	-Deve ser nomeado como verbo
	-camelCase
	Não existem metodos globais, devem ser definidos sempre em uma classe
